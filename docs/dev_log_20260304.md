# Myavatar 开发日志

## 2026-03-04 - Skill 与项目代码对齐

### 背景

之前 `.claude\skills\myavatar\SKILL.md` 中的内容与实际项目代码 (`C:/docker/Myavatar`) 存在较大差异：
- Skill 中描述的是虚构的 `tools/` 目录下的脚本
- 实际项目使用的是 LangGraph 状态图 + 节点架构
- Skill 中缺少大量实际项目中的配置项和功能说明

### 完成的工作

#### 1. 代码架构分析

深入分析了以下核心模块：

**状态图架构** (`src/orchestrator/`):
- `hybrid_graph.py` - 主状态图，支持视频输入流程和标准流程
- `state.py` - VideoGenerationState 状态定义，包含所有节点输入输出
- `graph.py` - 基础状态图实现

**节点实现** (`src/nodes/`):
- `n0_douyin_downloader.py` - 抖音视频下载
- `n0a_video_keyframe_extractor.py` - 关键帧提取
- `n0b_video_understanding_node.py` - 视频理解
- `n1c_hybrid_content_writer.py` - 混合内容写作
- `n2_slide_generator.py` / `n2b_hybrid_slide_generator.py` - 幻灯片生成
- `n3_browser_capture.py` - 浏览器捕获
- `n4_tts_synthesizer.py` - TTS 合成器
- `n5_ffmpeg_assembler.py` - 视频组装

**工具模块** (`src/utils/`):
- `marp_helper.py` - Marp 渲染
- `theme_manager.py` - 模板系统
- `tts_client.py` - CosyVoice 客户端
- `ffmpeg_mixer.py` - FFmpeg 组装 (OpenCV + FFmpeg 混合方案)
- `llm_config.py` - LLM 模型路由配置

**存储模块** (`src/storage/`):
- `provider_registry.py` - 提供商/模型路由注册表
- `persona_registry.py` - Persona 配置管理
