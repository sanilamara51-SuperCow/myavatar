# Myavatar 改造总计划（合并版）

## 1. 目标
- 从单次 CLI 流水线升级为桌面工作台 + 可观测生产流水线。
- 引入 CrewAI 稿件反思闭环，提升文稿质量并降低“粗暴感”。
- 支持多角色语音混剪，摆脱单一音色。
- 支持精致 PPT 模板系统，提升视觉一致性与可替换性。
- 引入浏览器自动截图编排（先截图，后续扩展录屏）。
- 支持多模型供应商配置，适配不同模型池与路由策略。

## 2. 关键产品决策（已锁定）
- 桌面框架：PySide6。
- 迭代范围：一期全量上线。
- 审稿策略：强制人工确认后再进入合成。
- CrewAI 机制：评分阈值 + 最大轮次（默认 3 轮）。
- 角色粒度：句级角色切换。
- 模板技术：Marp 主题包（不切 PPTX 引擎）。
- 浏览器抓图：由脚本输出 capture 指令自动编排。
- 抓图媒体：一期先截图，录屏后续扩展。
- 兼容策略：保持现有 CLI 兼容。
- 语速策略：默认自动稳态节奏（自动分句、停顿、测速回调）。
- 数据存储：本地 SQLite。
- 模型供应商配置：
  - 项目默认 + 节点可覆写
  - 密钥仅存环境变量引用
  - 一期覆盖文本/视觉/Crew 反思（TTS 暂不并入统一模型路由）

## 3. 架构增量
- 新增服务层：`src/services/`，统一提供流水线编排 API。
- 新增桌面端：`src/desktop/`，包含项目、素材、脚本、模板、角色、任务、结果面板。
- 新增数据层：`src/storage/` + SQLite（`workspace/.myavatar/app.db`）。
- 新增模型路由层：`src/utils/llm_config.py` + Provider Registry。
- 新增智能体层：`src/agents/`，承载 CrewAI 反思循环。

## 4. 数据与接口变更
- 扩展脚本结构：
  - 从 `voiceover: str` 升级到可分段结构（后续引入 `speaker_segments`）。
  - 增加抓图动作结构（后续引入 `capture_actions`）。
- Provider Registry 核心实体：
  - `ProviderProfile`
  - `ModelSpec`
  - `ProjectModelRoute`
  - `NodeModelOverride`
- 路由优先级：
  - 节点覆写 > 项目默认 > 系统默认（兼容旧 env）。

## 5. 第一阶段实施顺序
1. 合并规划文档并落库（本文件）。
2. 实现 Provider Registry（SQLite 表结构 + 路由解析 + 兼容回退）。
3. 将 N1/N1B/N1C 接入统一模型路由。
4. 增强运行日志，输出“节点实际使用供应商/模型”。
5. 补充基础校验与回归测试。

## 6. 中期实施顺序
1. CrewAI 反思链路 + 评分规约。
2. 审稿闸门（桌面端强制确认）。
3. 模板主题包系统（主题清单、切换、重渲染）。
4. 角色中心与句级角色混剪。
5. 语速治理与节奏指标可视化。

## 7. 后续实施顺序
1. 浏览器截图节点落地（n3）。
2. 失败重试与断点恢复。
3. 录屏能力扩展。
4. 更细粒度任务可视化与质量报表。

## 8. 验收标准
- 可在同一项目中为不同节点选择不同模型供应商与模型。
- 文稿在进入合成前必须经过人工确认。
- 支持句级多角色混剪并产出稳定音频。
- 模板可切换且视觉风格明显差异化。
- 自动抓图可回填素材并参与渲染。
- CLI 旧入口仍可正常运行。
